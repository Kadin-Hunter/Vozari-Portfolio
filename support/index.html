How can we help?

Start with the recovery steps below. If you still need help, send us the exact error message and a few details about your Mac.

Open a support request

Faster answers

What to include

Include your macOS version, Mac model, Vozari version, selected transcription mode, and the exact error message. GitHub issues are public. Do not include private audio, transcript content, or personal information.

Troubleshooting

Common recovery steps

01

Microphone unavailable

Open System Settings → Privacy & Security → Microphone, enable Vozari, then reopen the app.

02

Microphone changed during a meeting

Vozari stops safely and keeps every completed audio chunk. Select the input you want, then start recording again.

03

Source file moved

Choose the file again. Saved transcript text remains available even when its source audio is missing.

04

Model unavailable

Connect to the internet, retry the model download, and leave Vozari open until it completes.

05

Mac App Store dictation

After dictation finishes, press Command-V to paste the copied result.

06

Purchase not recognized

Open Settings → Vozari Pro → Restore Purchases.

07

Meeting system audio unavailable

Open System Settings → Privacy & Security → Screen & System Audio Recording, enable Vozari, then reopen the app. Vozari requests audio only even though macOS uses a screen-related permission name.

08

Interrupted meeting

Reopen Meetings. Vozari marks the session as recovered and lists every fully written 30-second chunk; use Send to Transcribe or delete the recording.

09

Disk became full

Vozari stops the meeting and preserves every completed audio chunk. Free storage space before starting another recording.
